# Анализ проектов через OpenAI

Приложение использует OpenAI API для формирования управленческих выводов и работы интеллектуального ассистента.

## Как это работает
1. Пользователь открывает карточку проекта.
2. Нажимает кнопку "Запустить анализ".
3. Backend собирает данные проекта и рассчитывает метрики (projectMetrics.ts).
4. Подготавливается компактный JSON для OpenAI.
5. Вызывается соответствующий ассистент OpenAI (идентификатор задается в `PROJECT_ANALYSIS_ASSISTANT_ID` или `CHAT_ASSISTANT_ID`).
6. Результат сохраняется в `data/analysis-results.json` и отображается в интерфейсе.

## Состав отправляемых данных
- Основные параметры проекта (название, сроки, стадия).
- Рассчитанные метрики (полнота данных, своевременность ПК, прогресс по весам).
- План/факт по кварталам.
- Список предупреждений.

## Конфигурация
Задайте следующие переменные в `.env`:
- `OPENAI_API_KEY` — общий ключ OpenAI, предоставляющий приложению доступ к OpenAI API. Конкретные ассистенты настраиваются через их ID, поэтому один API key используется для всех сервисов в рамках одного проекта OpenAI.
- `PROJECT_ANALYSIS_ASSISTANT_ID` — идентификатор ассистента для анализа карточки проекта (например, `asst_vHC5n2sz6bGbl1ldzlzR4OSQ`).
- `CHAT_ASSISTANT_ID` — идентификатор ассистента для работы чат-помощника (например, `asst_QYrPEDgjkF84OY9T6voYS9uz`).
- `CHAT_ASSISTANT_ENABLED` — включение или выключение чат-помощника (`true`/`false`).
