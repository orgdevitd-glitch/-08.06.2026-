# Интеграция с Google Sheets

Проектный Аналитик 2.0 поддерживает автоматический импорт и синхронизацию портфеля проектов из Google Sheets в режиме реального времени.

## Конфигурация переменной окружения

В настройках приложения (например, в переменных окружения панели Cloud Run) задается URL таблицы:

*   **Имя переменной**: `GOOGLE_SHEETS_CSV_URL` (или `GOOGLE_SHEET_CSV_URL`)
*   **Рекомендуемый формат**:
    ```text
    https://docs.google.com/spreadsheets/d/<ID_ТАБЛИЦЫ>/export?format=csv&gid=<ID_ЛИСТА>
    ```
*   **Альтернативный (человекочитаемый) формат**:
    ```text
    https://docs.google.com/spreadsheets/d/<ID_ТАБЛИЦЫ>/edit#gid=<ID_ЛИСТА>
    ```

> 💡 **Автоматическая нормализация**: Приложение автоматически распознает ссылки вида `/edit#gid=...` или `/edit?gid=...` и преобразует их в прямой CSV-экспорт с сохранением переданного идентификатора GID. Если идентификатор GID отсутствует, используется значение по умолчанию (`gid=0`), но в логи записывается предупреждение `Google Sheets gid was not provided, using gid=0`.

## Требования к доступу и совместному использованию

Для правильного сбора данных в контейнере Cloud Run таблица **должна быть открыта для чтения**:
1. Откройте ваш файл Google Sheets.
2. Нажмите кнопку **«Поделиться» / «Share»** в правом верхнем углу.
3. Измените общий доступ на **«Все, у кого есть ссылка, могут просматривать»** (General Access: Anyone with the link can view).

*Если доступа нет, Google Sheets вернет страницу авторизации (HTML-контент) вместо CSV-заголовков. Сервер приложения обнаружит HTML-ответ, запишет подробную ошибку и автоматически переключится на резервные данные (fallback).*

## Алгоритм работы и отказоустойчивость (Fallback)

1. **Режим Live**:
   При каждом запросе списка проектов бэкенд обращается к Google Sheets за актуальным CSV-файлом. Если загрузка и разбор проходят успешно, данные сохраняются локально (обновляя базу проектов, а также удаляя строки, отсутствующие в Sheets), и бэкенд возвращает статус `live`.

2. **Режим Fallback**:
   Если Google Sheets недоступен (ошибка сети, неверный URL, 403 Forbidden, неверные столбцы таблицы), приложение **не выдает ошибку пользователю** и **не очищает существующие данные**. Оно продолжает надежно работать на последней зафиксированной локальной копии (кэше) данных, возвращая статус `fallback` с подробным предупреждением.

## Обязательные колонки (Headers)

CSV-файл должен содержать на первой строчке обязательные столбцы для идентификации проектов:
*   `ID проекта` (или `ИД проекта`)
*   `Название проекта`

В случае их отсутствия, бэкенд логирует полный список найденных некорректных заголовков и активирует fallback-режим.

## Инструменты диагностики

Для удобного администрирования в интерфейсе предусмотрены:
- Индикатор **Live/Fallback** в заголовке рядом со временем синхронизации.
- Интерактивная кнопка **«Диагностика»** на предупреждающем баннере и индикаторе, открывающая детализированную панель параметров, URL, ответов сети и timestamps последних синхронизаций.
- Логи бэкенда с подробным разбором количества строк, структуры заголовков и ошибок скачивания.
